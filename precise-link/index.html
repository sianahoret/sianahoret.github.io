Would you like to be able to MAKE A HYPERLINK TO ANY TEXT ON A PAGE?


Currently browsers can resolve a reference to a certain place on a web page (by a "fragment identifier" in a URL). To exploit this feature the page author must provide a special mark at a certain place on a page and you must know how to use the mark to refer to that place. Often even if a page really contains such marks, it's not easy to find them (unless they are obviously shown to you with a hand/anchor/chain image etc.).
But what if you want to refer to an arbitrary part of the page, not marked anyhow?

Here are two very simple bookmarklets allowing you to make a link to a certain text on a page (a so named "precise link"), save or share it and later to open the link and see exactly the referred text shown and selected.

HOW TO USE:

  1. Drag and drop Make Precise Link bookmarket into bookmarks in your browser.
    Alternatively, if you want to share a precise link with not prepared people, you can use Make Self-explained Precise Link. This bookmarklet wraps a precise link in such a way that when you share it with somebody, the addressee will not need to know that this link is "precise" and what to do to resolve the link - he will be forwarded to an intermediate page containing necessary explanation.
  2. Drag and drop Resolve Precise Link bookmarklet into bookmarks in your browser.

Additional notes:

  • The text which you select before making a "precise link" should be as possible unique in the scope of the target page.
  • A "precise link" is the target page (where "Make Precise Link" bookmark was applied to) URL appended with an additional hash-parameter (#href-to-text=thetext).
  • It does not affect the shown page content and behaviour anyway.
  • When you visit the target page by a "precise link" and a browser does not have "Resolve Precise Link" bookmark (or you do not click the bookmark), the page will be shown exactly as if it was visited by its original URL.
  • No any web service is involved into URL processing during sharing/resolving - all happens in a browser.
  • No any permanent process is running - all happens only when you click a bookmark. BTW, every bookmarklet is just a few hundred bytes.

Alternatives:

In the past a few proposals and experimental solutions of the problem arose. Some of them are just wordings (proposals), others - browser-specific solutions, some stopped working for somw reasons. PreciseLink bookmarklets aim to be cross-browser (for now it works in all popular browsers except IE).
